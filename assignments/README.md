# Assignments — Web Platform & JavaScript Foundations

Each lesson has an assignment described in its `Lesson_NN.md`. Submit every assignment using `submission-template.md`, and back every claim with evidence from an instrument.

| File | Purpose |
|------|---------|
| `submission-template.md` | per-lesson submission format |
| `capstone-brief.md` | the full FORGE-9100 Web Diagnostics Capstone specification |
| `capstone-submission-template.md` | the capstone write-up format |

## What every submission must include
- **Investigation:** the instrument used, the evidence gathered, the layer localized.
- **Root cause** (not the symptom) and the **fix** that addresses it.
- **Verification:** measured before/after, a clean trace, passing tests, or correct status codes.
- **AI-usage log:** draft → verify-against-the-instruments → log.
- **Clean commits** (your Module 02 Git skills apply).

## Grading
Against `../ASSESSMENT_RUBRIC.md`. The recurring standard: *investigate first, optimize second*, and **never claim a cause you haven't observed.**
