# Resources — Web Platform & JavaScript Foundations

Reference material to use alongside the lessons and labs. These are working references, not reading assignments — keep the cheatsheets and the debugging playbook open while you work.

| Resource | Use it for |
|----------|-----------|
| `devtools-guide.md` | what each DevTools panel shows and how to read it |
| `javascript-runtime-reference.md` | types, coercion, scope, closures, `this` |
| `modern-js-cheatsheet.md` | ES6+ idioms (let/const, destructuring, modules, `?.`/`??`) |
| `async-event-loop-guide.md` | the event loop, promises, async/await, ordering rule |
| `performance-checklist.md` | load, runtime, data/algorithm, and rendering performance |
| `http-status-reference.md` | HTTP methods, status codes, REST design, fetch |
| `networking-fundamentals.md` | DNS → TCP → TLS → HTTP journey and the tools |
| `rendering-pipeline-guide.md` | layout/paint/composite, reflow, thrashing |
| `debugging-playbook.md` | evidence-first recipes for the common failures |
| `ai-workflow-guide.md` | draft → verify-against-the-instruments → log |
| `engineering-notebook-template.md` | the notebook structure you submit |

**The thread through all of them:** *investigate first, optimize second*, and **never claim a cause you haven't observed.**
